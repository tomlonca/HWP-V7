En el main se pregunta o se determina el rol de cada computadora. de ser posible por medio del make file.
Al compilar, se controla si se puso el nombre de un archiv. si se puso un nombre, se abre y sino, se lee directamente de un archivo llamado outputFile.

En la clase GeneralDriver se desarrollan las funciones principales que tienen que ver con el board
- leer
- escribir
- determinar rol

En la clase CalculateCRC se calcula la suma CRC. importante tanto para reader como writer

En la clase Reader se controla la reaccion de la computadora que lea. esto incluye:
- signalizar al writer que se esta listo para leer mediante FLAGS
- recibir datos
- en caso de calculo de CRC distinto al que dijo el writer, perdir de vuelta los datos
    -para eso hay que notificar a generalwriter que se intercambia el rol temporalmente para notificar el error
    -luego se reestablecen los roles y se repite la transaccion
-una vez controlados los datos, se guardan en un InputFile (opcional, se traducen)
-se envia una flag de que todo salio bien! (FIN DEL PROGRAMA)

En la clase Writer los datos otorgados en la main se van a escribir
-pedir resultado CRC de la clase CalculateCRC
-se envian flags para notificar que se esta listo
-se intercambian los roles brevemente para leer las flags de el reciever
-se reestablecen los roles y se envia una flag junto con la suma CRC
-luego de enviar eso se envia otra flag y se envian los datos
-cuando se terminan de enviar, se envia una flag de que se termino y se invierten los roles para ver la respuesta del reciever

*los datos se mandan con la suma CRC. De haber un error, el reciever le dice al sender que lo hay y los datos se piden de vuelta.
Se repite el proceso hasta que los datos lleguen correctamente.*
